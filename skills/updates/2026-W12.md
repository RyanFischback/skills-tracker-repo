## What This Tracker Is

This tracker captures the top skills used in real workflow execution and explains the practical value of each one. This week is an intentional single-skill focus on agent/subagent orchestration through `superpowers`, with subagents as the core theme.

## Skills This Week

1. `superpowers`: Uses a structured workflow (brainstorming -> planning -> Subagent Driven Development -> review and verification) so work is distributed to specialized agents instead of one long, fragile coding thread.

## Overall Benefit This Week

- Makes the lead agent an orchestrator and pushes implementation/review to subagents with clearer role boundaries.
- Improves reliability through a planning loop plus spec-review subagents before coding.
- Increases throughput with subagent parallelism and task-level handoffs.
- Improves cost/performance tradeoffs by routing tasks to the cheapest capable model.

## Why Subagents Are The Wave This Week

- Codex support for subagents makes this workflow immediately practical in day-to-day execution.
- The key upgrade is operational: one lead agent coordinates many focused workers instead of one overloaded thread.
- This creates better momentum on larger tasks because planning, implementation, and review can run as coordinated stages.

## How This Changes Agentic Programming

- Shifts from a single coding chat to a coordinated multi-agent system with explicit planner, implementer, and reviewer behaviors.
- Makes Subagent Driven Development the default path on harnesses that support subagents.
- Adds a spec review loop where a subagent checks planning docs for completeness before implementation starts.
- Reinforces decomposition discipline so larger projects can be executed as well-defined units.
- Keeps `Visual Brainstorming` as a supporting communication aid, not the core execution model.
